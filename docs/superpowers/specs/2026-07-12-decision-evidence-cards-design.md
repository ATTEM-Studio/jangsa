# 판단 근거 카드 디자인

## 목표

사장님이 `판단 근거 자세히 보기`를 열었을 때 숫자 목록을 해석하지 않아도, 현재 상황과 오늘 행동의 연결을 한 화면에서 이해하게 한다.

## 사용자 흐름

1. `현재 위치` 카드에서 목표까지 남은 매출·남은 기간·하루 필요 손님을 확인한다.
2. `숫자가 말하는 문제` 카드에서 선택한 고민에 관련된 수치와 쉬운 해석을 읽는다.
3. `그래서 오늘의 1순위` 카드에서 추천 행동이 앞선 이유를 확인한다.
4. `이번 추천의 기준` 카드에서 확정 입력값과 아직 검증하지 않은 가정을 구분한다.
5. `지금은 미루기` 카드에서 불필요한 행동을 피한다.

## 화면 요구사항

- 기존 details/summary 접기 구조를 유지한다.
- 정보는 카드 4개와 마지막 보류 카드로 표현한다.
- 각 카드에는 아이콘, 제목, 한 줄 해석, 필요한 숫자만 표시한다.
- `예상 효과`가 아닌 입력값·가정·확인 지표를 구분해 과장된 매출 약속을 피한다.
- 광고 추천에는 클릭 단가와 안전 클릭 기준을, 그 외 추천에는 매출 격차·하루 필요 손님을 표시한다.
- 가정이 없으면 `입력한 정보만으로 판단했습니다`를 표시한다.
- 모바일에서는 카드가 한 열로 쌓여 읽기 쉬워야 한다.

## 데이터와 구현 경계

- 추천 엔진의 행동 선택 규칙은 바꾸지 않는다.
- `src/app.js`의 근거 렌더 함수만 카드용 HTML을 생성한다.
- `index.html`은 근거 영역의 의미 있는 레이블만 보강한다.
- `assets/styles.css`는 카드, 흐름선, 상태 색상, 모바일 레이아웃을 담당한다.
- `tests/app.integration.test.cjs`는 샘플 진단 결과에 4개 핵심 근거 카드와 가정·보류 메시지가 존재하는지 검증한다.
