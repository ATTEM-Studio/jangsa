# 장사네비게이션 v12

장사네비게이션은 자영업자가 “오늘 무엇부터 고칠지”를 납득할 수 있게 보여주는 실행 코치입니다.

v12는 단순 추천 화면이 아니라, 링크 입력 → 사장님 확인 → 외부 매장 준비도 점수 → 내부 성장 병목 → 목표점수 → 오늘의 행동 → 7일 뒤 결과 확인으로 이어지는 진단 흐름을 제공합니다.

## 바로 실행하기

정적 버전은 `index.html`을 브라우저에서 열어 사용할 수 있습니다. 별도 서버 없이 동작하며, 입력과 이력은 현재 브라우저의 로컬 저장소에만 저장됩니다.

개발 검증은 다음 명령으로 실행합니다.

```bash
npm ci
npm run verify
```

## v12 진단 구조

- 외부 매장 준비도: 공개 정보와 사장님 확인값으로만 100점 계산
- 내부 성장 병목: 매출·객단가·광고·재방문을 점수에 섞지 않고 행동 판단에 사용
- 미확인 항목: 0점이 아니라 계산 제외
- 목표점수: 선택한 행동으로 회복 가능한 배점만 반영
- 업종 비교: 유효 표본이 준비되기 전에는 `권장 기준 대비`만 표시

현재 정적 버전은 수집기 인터페이스와 수동 확인 흐름을 제공합니다. 네이버 플레이스 자동 수집은 별도 서버 수집기 프로젝트에서 연결합니다.

## 사용자가 보는 핵심 흐름

1. 매장 링크를 입력하거나 수동으로 시작합니다.
2. 시스템은 자동 수집을 과장하지 않고, 확인 가능한 항목을 사장님에게 보여줍니다.
3. 사장님은 항목별로 좋음·보완·미흡·모름을 확인합니다.
4. 확인 범위가 충분하면 외부 매장 준비도 점수를 보여줍니다.
5. 매출 목표, 광고, 재방문, 수용 여력 등 내부 숫자는 “성장 병목” 판단에만 사용합니다.
6. 오늘 실행할 행동 하나, 하지 말아야 할 행동 하나, 확인할 지표 하나를 제안합니다.
7. 실행을 선택하면 7일 뒤 확인할 이력이 저장됩니다.

## 현재 범위와 한계

- 실제 네이버 플레이스 자동 수집, 계정 로그인, 서버 저장, 알림 발송은 포함하지 않습니다.
- 정적 버전의 이력은 사용자의 브라우저 로컬 저장소에만 남습니다.
- 매출 증가, 검색 순위 상승, 업종 내 상위 비율을 보장하지 않습니다.
- 유효 표본 데이터가 없는 업종 평균이나 퍼센타일은 표시하지 않습니다.
- 점수는 실행 우선순위를 돕는 참고값이며, 최종 판단은 사장님의 현장 상황과 함께 봐야 합니다.

## 검증

```bash
npm run verify
```

검증에는 점수 계산, 미확인 항목 처리, 60% 확인 범위 기준, 행동 우선순위, 공유 문구 민감정보 제거, 링크/수동 확인 흐름, 7일 확인 이력, 정적 빌드 테스트가 포함됩니다.

## 파일 구조

```text
index.html                 화면 구조
assets/styles.css          반응형 디자인
src/observations.js        확인값·출처·신뢰도 관찰 모델
src/storefront-score.js    외부 매장 준비도 점수 계산
src/action-priority.js     실행 가능성 기반 행동 우선순위
src/diagnosis-presenter.js 결과 표시·안전 공유 문구 모델
src/engine.js              내부 성장 병목과 추천 엔진
src/ui-logic.js            입력 검증·이력 모델
src/app.js                 화면 동작·로컬 이력
scripts/build-static.cjs   정적 빌드
tests/                     자동 검증
docs/validation-plan.md    실제 사용자 검증 계획
```

`dist/`는 빌드 결과이며 Git에는 올리지 않습니다.
